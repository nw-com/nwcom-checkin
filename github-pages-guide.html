🌐 GitHub Pages 設定完整指南

讓您的西北勤務管理系統快速上線！

🎯 開始之前： 請確保已經執行了快速部署腳本（quick-deploy.bat）
1

🔑 登入 GitHub 並找到您的倉庫

1. 前往 GitHub.com 並登入
2. 點擊右上角的頭像 → "Your repositories"
3. 找到您的 "nwcom-checkin" 倉庫並點擊進入

📱 截圖：GitHub 倉庫列表頁面
2

⚙️ 進入 Settings 設定頁面

1. 在倉庫頁面中，點擊頂部的 "Settings" 標籤
2. 如果找不到，確認您有足夠的權限
3. 滾動到頁面下方找到 "Pages" 選項

📱 截圖：Settings 選項位置
3

📄 找到 Pages 設定

1. 在左側選單中點擊 "Pages"
2. 您會看到 "GitHub Pages" 設定區域
3. 目前狀態應該顯示 "None" 或 "Not published"

📱 截圖：Pages 設定位置
4

🎯 設定部署來源

1. 在 "Source" 區域，點擊下拉選單
2. 選擇 "Deploy from a branch"
3. 在 "Branch" 下拉選單中選擇：
   • 通常選擇 "main"（新倉庫）
   • 或選擇 "master"（舊倉庫）
4. 文件夾選擇 "/ (root)"

Source: Deploy from a branch Branch: main Folder: / (root)
5

💾 保存設定

1. 點擊藍色的 "Save" 按鈕
2. 頁面會顯示 "Your site is ready to be published"
3. 記下提供的網址，格式如下：

https://[您的用戶名].github.io/[倉庫名稱]/
6

⏰ 等待部署完成

1. GitHub Pages 部署需要 2-5 分鐘
2. 您會看到綠色的成功訊息
3. 如果看到紅色錯誤，請檢查分支設定

✅ 您的網站已成功發布！
🎉 恭喜！ 您的西北勤務管理系統現在可以通過 GitHub Pages 訪問了！

✅ 完成檢查清單

💡 提示：
  • 如果 10 分鐘後仍無法訪問，請確認倉庫是 Public（公開）
  • 確保已經推送了代碼到 GitHub
  • 檢查分支名稱是否正確（main vs master）
